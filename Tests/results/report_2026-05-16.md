### Bottlenecks Identified

1. **External AI calls** (/notifications, /ai-insights): ~3-5s response time due to OpenRouter API
2. **DB schema mismatch** (/financial/predict): 100% failure - financial_risk table missing risk_label column
3. **DB write latency**: Prediction endpoints that write to DB take ~500-1000ms
4. **In-memory model inference** (predict_credit_risk): Fastest ML endpoint at ~7ms

## 3. Recommendations

1. Fix the DB schema mismatch for financial_risk and liquidity_risk tables (missing risk_label column)
2. Add caching for AI insight endpoints to reduce external API dependency
3. Consider async DB writes to improve throughput of prediction endpoints
4. Add connection pooling tuning for higher concurrency
5. Run a longer load test (5-10 min) with more users to find the true breaking point
